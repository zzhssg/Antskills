# S3 — 精修定稿

## 目标

根据用户反馈，将简版 Demo 升级为**完整 Demo**，确认所有视觉和交互细节后定稿。

## 方法论

**反馈驱动 + 三层补全。** 不猜用户意图，严格按反馈修改。

## 反馈解读规则

| 用户说 | 实际意思 | 行动 |
|--------|----------|------|
| "太多了" | 信息密度过高 | 折叠/隐藏次要信息 |
| "看不懂" | 结论不够直白 | 重写 Hero 文案，加颜色编码 |
| "不够" | 缺少某个维度 | 追问具体缺什么，加到论据层 |
| "丑" | 视觉风格不满意 | 追问参考，换色/换字体/换布局 |
| "OK" / "可以" | 定稿信号 | 确认后进入 S4 |
| "差不多" | 还有小问题 | 追问"差不多是哪里还不够？" |

## 三层补全清单

### Layer 1: Hero 补全
- [ ] 结论文案是否足够直白？
- [ ] 核心指标是否有颜色编码？
- [ ] 推荐排名是否有视觉权重区分（TOP 1 最大/最亮）？
- [ ] 默认状态的引导语是否有吸引力？

### Layer 2: 论据层补全
- [ ] 图表是否支撑 Hero 的结论？
- [ ] 交互是否流畅（切换输入 → 即时更新）？
- [ ] 对比维度是否足够？（至少 3 个维度）
- [ ] 异常数据是否有视觉标记？

### Layer 3: 信任层补全
- [ ] 方法论说明是否易懂？
- [ ] 风险提示是否与推荐对称？
- [ ] 原始数据是否可折叠查看？
- [ ] 数据来源是否标注？

## 迭代规则

1. 每轮反馈最多改 3 个点（避免改一个坏两个）
2. 改完后复述改动项，让用户确认
3. 如果反馈涉及架构级变更（如"换个完全不同的方式"），回到 S2 重做
4. 最多迭代 3 轮。第 3 轮后必须推动定稿或回到 S1 重新理解需求

## 定稿确认话术

> 这一版的变更：
> 1. [改动 1]
> 2. [改动 2]
> 3. [改动 3]
>
> 如果整体方向 OK，我下一步进入 PRD 阶段，把前端的每一个元素拆解为工程规范。确认定稿吗？
